2-Panel Login Page Redesign

Spec · 2026-05-20 · Status: Draft

Ringkasan

Ganti tampilan halaman login Frappe default di task.localhost:8080/login dengan desain 2-panel yang menarik: kiri gradient ungu-pink dengan feature list, kanan form login putih bersih. File implementasi: vernon_tasks/www/login.html.

Tujuan

  • Halaman /login (dan root / saat belum login) langsung menampilkan 2-panel login yang berkesan
  • Tidak ada halaman marketing perantara — user langsung disambut dengan login page
  • Desain konsisten dengan brand Vernon Tasks (purple gradient)

Desain Visual

Layout

  • Dua panel horizontal — kiri flex 1.35, kanan flex 1
  • Full viewport height (100vh)
  • Responsive: stack vertikal di layar < 768px (panel kiri jadi header)

Panel Kiri — Gradient Mural

  • Background: linear-gradient(145deg, #4f3cc9 → #7c3aed → #a855f7 → #f093fb)
  • Grid pattern overlay: repeating-linear-gradient putih opacity 5%
  • Blur blob dekoratif di sudut kiri atas dan kanan bawah
  • Logo Vernon Tasks (icon ◆ + nama) di atas
  • Headline besar: "Satu platform untuk seluruh alur kerja tim."
  • Subtext: penjelasan singkat 1 kalimat
  • Feature list (3 item dengan icon + nama + deskripsi):
    • 🎯 OKR Tracking — Hubungkan target organisasi ke task harian
    • 🏃 Sprint Board — Agile sprint dengan kanban visual dan visibilitas penuh
    • 🔄 PDCA Workflow — Struktur kerja Plan-Do-Check-Act yang terbukti efektif
  • Trust badge di bawah: "Sistem aktif · Vernon Corp" dengan green dot

Panel Kanan — Form Login

  • Background: #ffffff
  • Form terpusat, max-width 340px
  • Greeting: "Selamat datang 👋" + subtext
  • Field: Email/Username + Password (label uppercase kecil)
  • Tombol submit: gradient ungu → pink dengan box-shadow glow
  • Error state: red alert box dengan pesan dari API
  • Footer: copyright Vernon Corp

Implementasi Teknis

File yang dibuat

  • vernon_tasks/www/login.html — template Jinja override login Frappe
  • vernon_tasks/www/login.py — context script (opsional, untuk dev shortcuts)

Cara kerja override

Frappe mencari www/login.html di setiap installed app sebelum fallback ke default. File ini akan di-render sebagai Jinja template dengan context Frappe standar (csrf_token, redirect_to, dll).

Login API

POST /api/method/login
Content-Type: application/x-www-form-urlencoded

usr={email}&pwd={password}&cmd=login
  • Success: {"message": "Logged In"} → redirect ke /m/work atau ?redirect_to=
  • Error: non-2xx atau {"message": "..."} → tampilkan pesan error

No external CDN

Semua CSS inline dalam <style> block. Gunakan system font stack (-apple-system, BlinkMacSystemFont, 'Inter', 'Segoe UI', sans-serif). Tidak ada Tailwind atau external font request.

Dev shortcuts

Jika frappe.conf.developer_mode aktif, tampilkan quick-fill button "Administrator" di atas form untuk mempercepat development.

Out of Scope

  • SSO / OAuth — tidak diimplementasikan (tombol SSO/OTP di mockup dihapus dari implementasi)
  • Forgot password flow — gunakan link ke halaman Frappe default /update-password
  • Perubahan pada PWA login (pwa/src/auth/login.tsx) — tidak disentuh
  • Web Page fixture "home" — tidak diubah (Frappe redirect ke /login saat belum auth)

Testing

  • Buka http://task.localhost:8080/login saat logout → tampil 2-panel
  • Submit credentials valid → redirect ke /m/work
  • Submit credentials salah → error box muncul, form tetap aktif
  • Resize < 768px → stack vertikal, tidak ada overflow
