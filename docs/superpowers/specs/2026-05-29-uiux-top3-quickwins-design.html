UI/UX Top 3 Quick Wins

Date: 2026-05-29
Scope: Targeted UX fixes from 3-perspective review (UX Specialist, Project Manager, Team member)
Files affected: ProjectSidebar.tsx, ProjectFormModal.tsx (new), QuickAddTaskModal.tsx (new), MyWork/List.tsx, BottomNav.tsx, OfflineBanner.tsx, TaskActions.tsx

0. Background

Tiga reviewer (UX, PM, dev) memeriksa UI/UX PWA. Temuan yang konvergen di ≥2 reviewer dipromosikan ke prioritas tertinggi. Satu klaim dev ("CSS class hilang → build fail") diverifikasi salah — EmptyState pakai inline style + CSS var, build & test lulus. Spec ini mengeksekusi 3 quick win dengan dampak tertinggi terhadap core loop.

1. First-run CTA + create-project konsisten in-app

Masalah: ProjectSidebar saat kosong hanya menampilkan teks mati "Tidak ada proyek" tanpa CTA — user baru lihat layar buntu (risiko churn, PM P0 + UX P0). Selain itu MyWork "+ Proyek" memakai link Frappe eksternal (/app/vt-project/new) padahal sidebar sudah punya modal in-app — inkonsisten.

Solusi:

  • Ekstrak ProjectFormModal dari ProjectSidebar.tsx ke components/ProjectFormModal.tsx (shared). Sidebar import balik — tanpa perubahan perilaku.
  • Empty state ProjectSidebar: jika canWrite → tombol "+ Buat proyek pertama" membuka modal create; jika tidak canWrite → teks tetap.
  • MyWork "+ Proyek": ganti <a href external> jadi <button> yang membuka ProjectFormModal in-app. Setelah create, invalidate query proyek.
  • Link "Kelola" tetap eksternal — manajemen penuh memang berada di Frappe desk.

2. Quick-add task (modal + project picker)

Masalah: createTask hanya tersedia di dalam ProjectTaskPanel (harus pilih proyek dulu). Tidak ada quick-add dari Dashboard/MyWork — friksi inti produk task-management (PM P1).

Solusi: komponen baru components/QuickAddTaskModal.tsx.

  • Select proyek memakai hook useProjects: tampilkan title, kirim name. Tidak mengubah ProjectPicker lama (shape string[] berbeda).
  • Input judul task. Submit → createTask({ project, title }) via API vernon_tasks.api.projects.create_task (sudah ada di Project/api.ts).
  • Telemetry: quick_add_task_open, quick_add_task_submit.
  • Trigger: tombol di header MyWork (di samping "+ Proyek").
  • Bila belum ada proyek: modal menampilkan pesan arahkan buat proyek dulu (tidak ada select kosong yang membingungkan).

4. Testing

  • Reuse pola test: ProjectSidebar.test.tsx, components.test.tsx, List.test.tsx.
  • Kasus baru:
    • ProjectSidebar empty + canWrite → render tombol CTA; klik buka modal.
    • ProjectFormModal shared: create memanggil createProject, edit memanggil updateProject.
    • MyWork "+ Proyek" membuka modal in-app (bukan navigasi eksternal).
    • QuickAddTaskModal: submit memanggil createTask dengan {project, title}; tombol disable saat title kosong.
    • BottomNav: fontSize 12 & minHeight 48 ter-render.
    • OfflineBanner: bg danger saat offline.

5. Out of scope (v2)

  • Sprint kanban mobile (drag antar kolom)
  • Leader bulk actions / inline edit
  • Offline conflict resolution UI + sync queue count
  • Dark mode prefers-color-scheme di mobile
  • Refactor TopNav.tsx 629-line, komponen Badge/Modal shared (utang teknis, bukan blocker ship)

6. Klaim ditolak (verifikasi gagal)

Dev reviewer: "CSS class empty-state/page-skeleton hilang → build fail". Salah. EmptyState memakai inline style + CSS var (var(--vt-space-6) dll), className hanya dekoratif. Build & test lulus (dikonfirmasi git log). Tidak ditindaklanjuti.
