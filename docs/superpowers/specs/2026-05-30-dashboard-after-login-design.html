Dashboard Page After Login

Date: 2026-05-30
Task size: M (new desk Page + 3 www edits + fixture + css, ~8 files)
App ownership: 100% vernon_tasks — no Frappe core files touched
Scope: New presentation-only desk Page consuming existing whitelisted APIs; login redirect repoint

1. Goal

After a user logs in at task.localhost:8080/login, land them on a modern dashboard page (/app/vt-home) instead of the bare Frappe desk (/app). The page surfaces personal progress, projects, schedule, and quick links — reusing the rich APIs that were built for the now-removed PWA and have been orphaned since.

2. Design Decisions

4. Redirect Mechanism

Login flow has three repoint points. All are app-owned overrides of Frappe core www — core is never modified.

PointFile:lineChange
Server default (guest login)www/login.py:12redirect_to = ... or "/app/vt-home"
Client fallback (post-login JS)www/login.html:525(redirect_to or "/app/vt-home") — fixes stale /m/dashboard
Already-authenticated redirectwww/index.py:10redirect_location = "/app/vt-home"

The ?redirect_to= query param is still honored and validated against open-redirect (login.py:13-14: must start with /, not //). Route /app/vt-home is auto-registered by Frappe from page_name.

5. Layout (inside desk chrome)

Rendered into page.main via frappe.ui.make_app_page({single_column: true}). Each block is its own render function (≤ 40 lines).

  1. Hero band — eyebrow (uppercase tracking), serif greeting "Selamat datang, Nama", and 3 workload number cards (open / overdue / due soon) from me_progress.workload. Grid pattern + blue radial glow background.
  2. My Progress — velocity bar chart (8 weeks, frappe.Chart), active sprint progress bar + risk chip, next actions list (5) from me_progress.
  3. My Projects — led-project cards (progress, blockers, risk chip) + member table from my_projects.
  4. Schedule — 8-day agenda grouped per day (Today / Tomorrow / weekday) from schedule_agenda.
  5. Quick links — chips/buttons to VT Task / VT Project / VT Sprint (list + new) via vt_render_page_nav.

6. Visual Convention (from login)

Scoped under .vt-home so desk styles are not affected globally.

  • Fonts: DM Sans (body), DM Serif Display (headings/greeting, italic accent) — loaded via Google Fonts @import in vt_home.css
  • Palette: --navy #0b0f1a, --navy-2 #111827, --blue #2563eb, --blue-dk #1d4ed8, slate text #64748b/#94a3b8, border #e2e8f0, success dot #22c55e
  • Components: eyebrow label (10px, letter-spacing 3px, uppercase, blue), serif headline with muted italic span, chip with blue dot, cards (radius 6px, slate border), fadeUp staggered entrance animation
  • Risk chips: on_track = green, at_risk = amber, behind = red (named constants in JS)

7. Layer & Quality Rules

  • JS = handler layer: fetch API → render DOM. No business logic, no calculations beyond formatting.
  • All colors, labels, limits = named constants at top of vt_home.js.
  • Each render function ≤ 40 lines; file split per section.
  • No new whitelisted methods — reuse dashboard.py (already has require_login).
  • Role gating via Page roles in JSON (VT Member/Leader/Manager; System Manager has implicit access).

8. Testing

  • [ ] dashboard.py API unit tests still green (existing coverage).
  • [ ] /app/vt-home route reachable for logged-in VT Member.
  • [ ] Guest hitting / → /login; after login → /app/vt-home.
  • [ ] Stale /m/dashboard fallback removed from login.html.
  • [ ] Role without VT role cannot open the page (Frappe permission gate).
  • [ ] All 5 blocks render with real data; empty states handled (no sprint, no projects, empty agenda).
  • [ ] Fonts + palette match login; fadeUp animation plays; responsive at narrow width.

9. Fixture & Deploy

Add to hooks.py fixtures: {"dt": "Page", "filters": [["name", "=", "vt-home"]]}. After implementation: bench --site task.localhost migrate (registers Page), bench build --app vernon_tasks (bundles css/js), then verify in browser.
