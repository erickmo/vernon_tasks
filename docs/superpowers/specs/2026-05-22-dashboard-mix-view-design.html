Dashboard Mix-View Redesign

PWA /m/dashboard — Me / Projects / Schedule tabs
Date: 2026-05-22 Spec: docs/superpowers/specs/2026-05-22-dashboard-mix-view-design.html

1. Goal

Replace current single-page /m/dashboard (stats trio + active sprint + kanban scroll) with a three-tab mix-view dashboard. User wants to see personal progress, project progress, and schedules in one place, with role-aware scoping (admin sees all projects).

2. Scope

  • Full replacement of pwa/src/mobile/pages/Dashboard.tsx.
  • New folder pwa/src/mobile/pages/Dashboard/ with layout + 3 tab routes.
  • 3 new backend whitelisted endpoints in vernon_tasks/task/api/dashboard.py.
  • Telemetry events for each tab + key interactions.
  • Google Calendar integration: design slot only, off behind feature flag, not implemented v1.

3. Routes & Layout

/m/dashboard            → redirect to /m/dashboard/me
/m/dashboard/me         → MeTab
/m/dashboard/projects   → ProjectsTab
/m/dashboard/schedule   → ScheduleTab

Shared DashboardLayout renders hero (date + greeting), sticky segmented tab strip (TopNav pill style), and <Outlet />. Tab strip shows badge counts:

  • Me badge = workload.overdue
  • Projects badge = count of projects with risk ∈ {at_risk, behind}
  • Schedule badge = today_summary total (tasks + meetings + sprint_events)

4. Me Tab

Block A — Velocity strip

  • Inline SVG sparkline, last 8 weeks of tasks closed per ISO week.
  • Right side: this-week count + delta vs previous week (12 ↑3 green / 8 ↓2 red / neutral).

Block B — Sprint commitment card

  • Active sprint name + dates.
  • Stat row: committed X pts · done Y pts · Z%.
  • Progress bar (purple→indigo gradient, reused from current Dashboard).
  • Risk pill rule: elapsed_pct > 60 && done_pct < 50 ⇒ "Behind"; else "On track".
  • Empty state: EmptyState "Belum ada sprint aktif" when sprint null.

Block C — Workload chips (3-up grid)

  • Open: assigned, not done — INDIGO.
  • Overdue: deadline < today, not done — RED if >0 else dim.
  • Due ≤ 3d: deadline within 3 days, not done — AMBER if >0 else dim.

Block D — Next actions list

  • Top 5 tasks assigned to me, sorted priority desc, deadline asc.
  • Row: title · project badge · deadline chip · priority dot.
  • Tap → /m/task/{id}. Footer link "See all" → /m/my-work.

5. Projects Tab

5.1 Role gating

  • Admin: frappe.session.user has role System Manager OR Vernon Admin ⇒ scope = all projects in org.
  • Non-admin: scope = projects where I am lead OR member.

5.2 Filter strip (sticky)

Segmented: [All] [Led by me] [Member] [At risk].

  • Non-admin: All = led + member.
  • Admin: All = every project; Led by me still narrows.

5.3 Led projects — full detail cards

Card content:

  • Project name + status badge (Active/Paused/Done).
  • Active sprint name + 7-day burndown mini-chart (ideal vs actual lines, inline SVG).
  • 3 inline stats: % done, open tasks, blockers.
  • Risk flag pill: On track / At risk / Behind.
    • At risk = blockers > 0 OR (overdue + due_soon) > 2
    • Behind = sprint behind rule (same as Me Block B)
    • else On track
  • Tap → /m/project/{id} (route TBD; out of scope; leave hook).

5.4 Member projects — compact rows

One row per project I am member but not lead. Row: project name · % done · next milestone date · my open task count. Tap → same project route.

5.5 Admin variant

Admin sees Section 5.3 layout for ALL projects (every project rendered as full-detail card). Section 5.4 hidden when admin.

6. Schedule Tab

6.1 Horizon & sources

Today + next 7 days (8-day window). Sources for v1:

  1. Tasks assigned to me with deadline in window — type=task.
  2. Sprint events for sprints I'm in — start_date, end_date in window — type=sprint_start / sprint_end.
  3. Vernon Meetings if Meeting doctype exists at impl time — type=meeting. If absent: skip silently.

6.2 Layout

  • Today chip-row: Tasks due (n) · Meetings (n) · Sprint events (n). Tap scrolls/filters list.
  • Day-grouped list: sections Today, Tomorrow, then Day Name, DD MMM for days +2..+7. Empty days hidden.
  • Item row: optional time chip (meetings; tasks show "EOD" if deadline date-only) · type icon (task=◈, meeting=●, sprint=◆) · title · project badge.
  • Tap → respective detail route.

6.3 Sort within day

time asc, priority desc, title asc.

6.4 Google Calendar slot

Bottom of tab: muted card "Hubungkan Google Calendar" + disabled CTA. Behind feature flag feature.google_calendar (default off). Endpoint accepts ?include=google; returns empty array for now.

7. API Contracts

File: vernon_tasks/task/api/dashboard.py. All endpoints whitelisted, return JSON. React Query staleTime 60s.

@frappe.whitelist()
def me_progress():
    return {
        "velocity": [{"week": "2026-W17", "done": 8}, ...],   # 8 entries
        "velocity_delta": 3,                                   # vs prev week
        "sprint": {                                            # or null
            "name": "Sprint 12",
            "start_date": "2026-05-15",
            "end_date":   "2026-05-28",
            "committed_points": 21,
            "done_points":      13,
            "progress_pct":     61.9,
            "risk":             "on_track" | "at_risk" | "behind",
        },
        "workload": {"open": 14, "overdue": 2, "due_soon": 5},
        "next_actions": [
            {"id": "TASK-001", "title": "...", "project": "P1",
             "deadline": "2026-05-23", "priority": "High"},
            ... up to 5
        ],
    }

@frappe.whitelist()
def my_projects(filter: str = "all"):
    # filter ∈ {all, led, member, at_risk}
    return {
        "is_admin": bool,
        "led":    [ProjectCard, ...],
        "member": [ProjectRow, ...],  # empty if admin
    }
# ProjectCard = {id, name, status, sprint{name,start,end,burndown_ideal[],burndown_actual[]},
#                pct_done, open_tasks, blockers, risk}
# ProjectRow  = {id, name, pct_done, next_milestone, my_open_tasks}

@frappe.whitelist()
def schedule_agenda(include: str = ""):
    # include csv: 'google' reserved, ignored in v1
    return {
        "today_summary": {"tasks": 2, "meetings": 1, "sprint_events": 0},
        "days": [
            {"date": "2026-05-22", "label": "Today",
             "items": [AgendaItem, ...]},
            {"date": "2026-05-23", "label": "Tomorrow", "items": [...]},
            ...
        ],
    }
# AgendaItem = {type, id, title, project, time|null, priority|null, route}

9. Testing

9.1 Backend (pytest)

  • test_me_progress: velocity window math, commitment %, risk-rule boundary cases (60% / 50% edges), workload counts.
  • test_my_projects: filter scopes, admin bypass via System Manager, member-only user receives compact rows.
  • test_schedule_agenda: 8-day window edges, within-day sort, sprint event inclusion, graceful skip when Meeting doctype absent.

9.2 Frontend (Vitest + RTL)

  • MeTab.test.tsx: skeleton→data render, risk pill toggle at boundary, empty-state when sprint null.
  • ProjectsTab.test.tsx: filter switching, admin variant hides member block, at-risk filter logic.
  • ScheduleTab.test.tsx: day grouping, today chip-row counts, Google card hidden behind flag.
  • Dashboard/index.test.tsx: /m/dashboard redirects to /m/dashboard/me, tab badges render counts.

10. Out of scope

  • Google Calendar integration (slot only).
  • /m/project/{id} detail page (tap target hook leaves it for follow-up).
  • Drill-down weekly breakdown from velocity sparkline.
  • Server-side caching (React Query 60s staleTime is sufficient v1).

11. Risks / open questions

  • Meeting doctype: existence unknown at spec time. Impl must detect and skip gracefully — covered in test.
  • Vernon Admin role: may not exist yet. If absent, only System Manager grants admin scope.
  • Burndown data: requires per-day point snapshots. If history table missing, render straight ideal line + actual point only (degraded mode).
