www-react Dashboard — Design Spec

Standalone desktop SPA on its own domain, separate from existing PWA (/m/*) and Portal (/portal/*).
Date: 2026-05-23 Author: Erick Mo (with Claude) Status: Draft — pending user review Source: brainstorming session 2026-05-23

Goal

Ship a new desktop dashboard SPA www-react for all roles (IC, Leader/PM, Exec). Hosted on a separate domain via Caddy reverse-proxy to the existing Frappe backend. Coexists with existing /m/* PWA and /portal/* desk portal — does not replace them.

Five pages: Login, Dashboard (me + my projects), Projects (list + detail), My Worksheet (weekly planner), Reports.

Non-Goals

  • Replacing the existing /m/* PWA or /portal/* desk portal.
  • Mobile-first design — this is desktop-first; mobile breakpoints are graceful degradation only.
  • Offline support (no Service Worker, no IndexedDB).
  • Real-time collaboration (no websockets).
  • Building a new auth system — reuses Frappe session cookie.

Advisory Panel Consulted

  • UX specialist — sidebar layout, F-pattern dashboard, command palette, anti-modal-chain
  • COO — dashboard metrics that drive ops decisions, reports ≠ dashboard #2, fewer-touches principle
  • Project Manager — list columns, bulk actions, todo grouping toggle, team-worksheet privacy

Hosting

  • Folder: apps/vernon_tasks/www-react/ (sibling of pwa/)
  • Build output: www-react/dist/
  • Caddy reverse-proxy: dashboard.<domain> serves static dist/; /api/* proxied to Frappe bench
  • Frappe CORS: allow_cors + Access-Control-Allow-Credentials: true; session cookie set SameSite=None; Secure

Auth Flow

  1. /login POSTs api/method/login with (usr, pwd) — Frappe sets sid cookie cross-domain
  2. All subsequent requests use credentials: 'include'
  3. 401 interceptor → clear React Query cache → redirect /login?next=<path>
  4. Logout → api/method/logout + cache clear

2. Dashboard /portal/dashboard

Principle (COO): glanceable, decide-and-act. Not sticky. Time-in-app = cost, not value.

Layout (3 zones, F-pattern)

┌──────────────────────────────────────────────────────────┐
│ AT-RISK BANNER (red, conditional)                        │
│ "3 projects at risk · Alpha -12pts WoW · Beta overdue"   │
├─────────────────────────────┬────────────────────────────┤
│ TODAY (top-left, primary)   │ ME (top-right)             │
│ • PDCA queue count by phase │ • points week / streak     │
│ • next deadline (sprint/KR) │ • capacity used %          │
│ • blocked tasks owned (CTA) │ • on-time rate 7d          │
├─────────────────────────────┴────────────────────────────┤
│ MY ACTIVE SPRINTS — horizontal scroll cards              │
│ [Sprint card: name, burndown spark, days_left, %done] ×N │
├──────────────────────────────────────────────────────────┤
│ MY PROJECTS GRID — cards                                 │
│ [name, health dot R/Y/G, OKR bar, my role, blocked cnt]  │
└──────────────────────────────────────────────────────────┘

At-Risk Banner Trigger

  • health drop >10pts WoW, OR
  • ontime_rate <70%, OR
  • no check-in >5d on owned project, OR
  • overdue Key Result with flat progress

Click drills to project detail. Dismiss = 7d snooze (per user, per project).

Today Card Metrics

  1. on-time completion rate 7d rolling
  2. blocked/overdue count (clickable → filtered Tasks view)
  3. OKR confidence delta WoW

Explicitly Excluded (Anti-Vanity)

  • hours logged this week
  • total task count
  • team-wide stats (those belong on Reports)

Role Adaptation

  • IC: Today + Me + own sprints + own projects
  • Leader/PM: + team-blocked surface in banner; team capacity sparkline replaces personal capacity tile
  • Exec: + org-wide Health Score tile replaces Today; drill-down opens Reports

API

  • task.api.portal_dashboard.get_home(role) → single bundled payload, 1 round-trip
  • TanStack Query: staleTime 60s, refetchOnWindowFocus: true

  • Sort: health asc · days_left asc · blocked desc

Row Actions

Bulk Actions (checkbox column, slide-in action bar)

  1. Move tasks → next sprint
  2. Reassign owner
  3. Phase shift (BACKLOG→PLAN batch)

breadcrumb · name · health pill · pin · share · settings menu

  • KR group header: KR title, target/current bar, contributing tasks count
  • Row: title, PDCA pill, assignee avatar, due date, points, risk flag, inline status dropdown

full-size chart, ideal vs actual, forecast band. Velocity sparkline last 5 sprints. Sprint dropdown selector.

API

4. My Worksheet /portal/worksheet

Principle (COO): view, not form. Auto-pull tasks, no re-entry. Drives Friday review.

Layout

┌────────────────────────────────────────────────────────────┐
│ WEEK HEADER: « W21 May 18–24 2026 »  [Today] [This Week▼] │
│ Capacity used: 28h / 40h ████████░░  · OKR coverage: 80%   │
├──────────────┬────┬────┬────┬────┬────┬────┬────┬─────────┤
│ UNSCHEDULED  │Mon │Tue │Wed │Thu │Fri │Sat │Sun │         │
│ (right tray) │ 18 │ 19 │ 20 │ 21 │ 22 │ 23 │ 24 │         │
│              │08 ▓│08  │08  │08  │08  │    │    │         │
│ [task card]  │09 ▓│09 ▓│09  │09  │09  │    │    │         │
│ [task card]  │10  │10 ▓│10 ▓│10  │10  │    │    │         │
└──────────────┴────┴────┴────┴────┴────┴────┴────┴─────────┘

Source Data (auto-pulled)

  • All my tasks where status ∈ {PLAN, DO, CHECK} this week + next week
  • Capacity from vt_employee_capacity (daily hours)
  • Scheduled tasks read from vt_task_schedule_entry (date + hour_start + hours_planned)

Drag-Drop (dnd-kit)

  • From Unscheduled tray → day slot creates vt_task_schedule_entry
  • Between days updates date on entry
  • Vertical resize adjusts hours_planned
  • Optimistic update + undo toast (5s, sonner)

Capacity Visualization

Per-day bar at bottom of column: filled/capacity. Green ≤80%, amber 80–100%, red >100%. Week total in header.

Task Card Content

Project chip (color-coded), title (1 line, truncate), PDCA pill, points, linked KR icon. Click opens side panel detail (no modal chain).

View Toggles

  • This Week (default, weekly grid)
  • Today (focus mode: hour blocks 8–18 single column, large cards)
  • Next Week (planning ahead)

Friday Review Mode

Banner auto-suggests Friday 15:00+: "Wrap up week: 6 tasks done, 2 carry-over, log retrospective notes". One-click bulk-move incomplete → next week, append note.

Empty Slot Click

Quick-add: pick from Unscheduled tray, or create new task linked to project.

Team View Toggle (PM/Leader only)

  • Top-right toggle: "Personal | Team"
  • Team view is read-only aggregate grid: rows=members, cols=days, cells=hours + task count
  • No content peek — only capacity figures (PM panel privacy verdict against micromanagement)
  • Highlight overload (>100% capacity) red

API

  • task.api.portal_worksheet.get_worksheet(week_start) → tasks + schedule_entries + capacity
  • task.api.portal_worksheet.schedule_task(task_id, date, hour_start, hours)
  • task.api.portal_worksheet.unschedule(entry_id)
  • task.api.portal_worksheet.bulk_carry_over(week_start) — Friday batch
  • task.api.portal_worksheet.get_team_worksheet(week_start) — PM only, aggregate hrs only

Anti-Pattern Guard

NO manual time-tracking form in Worksheet. Actual hours logged via Task slide-over (reuse PWA pattern). Worksheet schedules planned hours only.

  • Toolbar right: Refresh · Export CSV · Export PDF · Schedule · Permalink
  • Visualization panel (recharts)
  • Data table below (sortable, paginated, virtualized >100 rows)
  • Narrative side panel: auto-generated 3–5 bullet insights (e.g., "Project Alpha health dropped 12pts WoW driven by 4 overdue tasks")

Three COO-Mandated Reports — Detail

project-health Heatmap

  • Rows = projects, cols = last 8 weeks
  • Cell color = health bucket (R/Y/G)
  • Click cell → drill to /portal/projects/:id at that week
  • Export: CSV per-week scores + trend arrow

okr-pacing Pacing Gap

  • Per KR: pace_expected (linear elapsed) vs current_progress
  • Gap bar: green if ≥0, red if <0
  • Group by Objective
  • Sort by gap asc (worst first)

team-throughput

  • Two lines: velocity (points/week), avg cycle time (hrs PLAN→DONE)
  • Segment toggle: by squad / by individual
  • Sprint boundary annotation markers

Export

  • CSV: client-side (papaparse) for ≤5k rows; server endpoint portal_reports.export for larger
  • PDF: server-side via Frappe print format → wkhtmltopdf
  • Schedule: new doctype vt_report_subscription (cron, email, recipients) behind feature flag portal_report_schedule_enabled

API

  • task.api.portal_reports.list_reports(role)
  • task.api.portal_reports.run_report(slug, filters)
  • task.api.portal_reports.export(slug, filters, format)

Guards

  • No live polling — reports = snapshot, not realtime
  • Server-side cache 5min per (slug, filters) key
  • Rate-limit export: 10/min per user

6. Cross-Cutting Concerns

Data Flow

  • TanStack Query is the single source of truth for server data (no Redux)
  • Query keys: ['dashboard'], ['projects', filters], ['project', id], ['project', id, 'tasks', groupBy], ['worksheet', weekStart], ['report', slug, filters]
  • Mutations: optimistic update + invalidate parent key + undo toast (5s window)
  • Zustand slices: UI-only (sidebar collapsed state, filter chips, command palette open, theme)

Error Handling

  • Global axios interceptor: 401 → clear cache + redirect /login?next=<path>; 403 → toast "Permission denied"; 5xx → toast + Sentry capture; network fail → retry 2× with backoff, then offline banner
  • Per-route error boundary (React Router errorElement)
  • Empty state: icon + headline + CTA
  • Loading: content-shape skeleton, debounced 200ms (not spinner)

Performance

  • Code-split per route (React.lazy + Suspense)
  • Route prefetch on sidebar hover
  • Table virtualization (@tanstack/react-virtual) for >100 rows
  • Bundle target: initial JS ≤180 KB gzipped (Vite manualChunks: react, query, charts, ui)

Accessibility

  • All interactive elements keyboard reachable, focus-visible ring
  • shadcn primitives wrap Radix UI (a11y built-in)
  • Color not sole indicator: health pill = dot + letter (R/A/G + ✕/!/✓)
  • WCAG AA contrast checked in design tokens

Internationalization

  • i18next, locale = Frappe user language on login boot
  • Initial bundles: id + en
  • Dates via date-fns + locale
  • Numbers via Intl.NumberFormat

Theme

  • shadcn dark + light, default = system
  • User choice persisted in Frappe user_preferences doctype
  • Brand accent #6836a0 (matches Vernon brand)

Security

  • CSP via Caddy: default-src 'self'; connect-src 'self' https://<api-domain>; img-src 'self' data:;
  • Frame-Ancestors: 'none'
  • Session cookie cross-domain only — no localStorage tokens
  • XSS: React default escape + DOMPurify on markdown render (KR notes, activity stream)

Efficiency Wins (UX panel)

  • Cmd+K command palette: jump project, create task, log PDCA, open report
  • List keyboard nav: J/K navigate rows, E edit, C create, / focus search
  • Optimistic drag-drop with undo toast
  • Inline edits everywhere — no modal chains
