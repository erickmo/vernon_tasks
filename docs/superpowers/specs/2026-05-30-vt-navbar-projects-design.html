Editable Global Navbar + Projects Cards Page

Date: 2026-05-30
Task size: L (new child doctype + boot hook + new desk Page + global nav script + css + hooks)
App ownership: 100% vernon_tasks — no Frappe core files touched
Scope: A second navigation bar shown on every desk page, with menu items editable from VT Settings; plus a new vt-projects desk Page that lists projects as cards.

1. Goal

Add a global "navbar2" under the desk navbar with menu links (default: Home → /app/vt-home, Project → /app/vt-projects). The menu must be editable by an admin without code, via a child table on VT Settings. Also add the vt-projects page that renders the user's projects as cards.

2. Design Decisions

DecisionChoiceWhy
Where menu items are storedChild table navbar_items on the existing single doctype VT Settings, rows of new child doctype VT Navbar ItemNative low-code config; single source; admin (VT Manager) edits in the VT Settings form. No code change to add/reorder links.
How items reach the browserFrappe extend_bootinfo hook injects bootinfo.vt_navbar_itemsNative boot mechanism; no extra HTTP round-trip; available synchronously to desk JS.
Out-of-box behaviourBoot function returns a hardcoded default (Home, Project) when the table is emptyWorks immediately on a fresh install; still fully overridable by editing the table.
Renderingpublic/js/vt_navbar.js via app_include_js; styles .vt-navbar2 appended to vt_home.css (already global)One global include; reuses the existing app_include_css file.
Projects page dataNew Page vt-projects reusing dashboard.my_projectsAPI already returns card-ready data (name, status, pct_done, open_tasks, blockers, risk). No new business logic.

4. Data Model — VT Navbar Item (child)

FieldTypeNotes
labelDatareqd; display text
routeDatareqd; desk route, e.g. /app/vt-home
iconDataoptional; Frappe icon name (e.g. home)
enabledCheckdefault 1; only enabled rows render

VT Settings gets a Table field navbar_items (options = "VT Navbar Item") under a new "Navbar Menu" section. Row order (idx) drives display order.

5. Boot Injection

hooks.py: extend_bootinfo = "vernon_tasks.boot.extend_bootinfo".

DEFAULT_NAVBAR = [
    {"label": "Home", "route": "/app/vt-home", "icon": "home"},
    {"label": "Project", "route": "/app/vt-projects", "icon": "folder-normal"},
]

def extend_bootinfo(bootinfo):
    # Inject the editable navbar2 menu so desk JS can read frappe.boot.vt_navbar_items.
    rows = frappe.get_all(
        "VT Navbar Item",
        filters={"parent": "VT Settings", "enabled": 1},
        fields=["label", "route", "icon"],
        order_by="idx asc",
    )
    bootinfo.vt_navbar_items = rows or DEFAULT_NAVBAR

6. Global Renderer (vt_navbar.js)

  • Reads frappe.boot.vt_navbar_items (fallback to an inline default mirroring DEFAULT_NAVBAR if the boot key is absent).
  • Builds a single .vt-navbar2 bar inserted once after the desk navbar (guard via a DOM id check to avoid duplicates on route changes).
  • Highlights the active item by comparing frappe.get_route_str() / window.location.pathname against each item route prefix.
  • Re-evaluates active state on frappe.router.on("change", ...).
  • Click navigates via frappe.set_route(route) (handles /app/... absolute paths in v15). All labels escaped with frappe.utils.escape_html.

7. Projects Cards Page (vt-projects)

  • Page vt-projects, title "Proyek", roles VT Member/Leader/Manager.
  • JS calls vernon_tasks.task.api.dashboard.my_projects; renders led projects + member projects as .vh-card grid (reuses vt_home.css classes — already global via app_include_css).
  • Each card: title, status, percent-done bar, open_tasks, blockers, risk chip; click → frappe.set_route("vt-project", id).
  • Empty state when no projects.

8. Layer & Quality

  • boot.py is a framework-event hook (Priority 3) — reads config only, no mutation.
  • JS = presentation only; named constants for defaults/colors; functions ≤ 40 lines; labels escaped.
  • No new whitelisted methods (reuse my_projects). No Frappe core edits.
