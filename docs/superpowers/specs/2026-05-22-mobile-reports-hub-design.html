Mobile Reports Hub Design Spec

Date: 2026-05-22 Parent: docs/superpowers/specs/2026-05-18-portal-reports-p5-design.html Status: Draft — pending review Owner: Erick Mo

1. Scope

Mobile /m/reports landing page with 3 role-filtered cards, drilling into category-specific reports. Replaces /m/analytics (redirect). Backend hybrid: reuse vernon_tasks/api/portal_reports.py for projects/team, reuse vernon_tasks/api/analytics.py for "My Reports". Role gating: Member sees only "My Reports"; Leader+ sees all three.

In scope:

  • Routes: /m/reports, /m/reports/me, /m/reports/projects, /m/reports/projects/:id, /m/reports/team.
  • Redirect /m/analytics → /m/reports/me.
  • BottomNav swap label Analytics → Reports.
  • Role-filtered landing (hide cards without access).
  • Feature flag mobile_reports_enabled (Check, default 0) in VT Settings.
  • One new whitelisted endpoint: list_managed_projects in portal_reports.py.

Out of scope:

  • New backend services (reuse existing).
  • Drill into individual team member.
  • Export PDF/CSV.
  • Custom date pickers beyond preset period chips.
  • New schema (no VT Team doctype).

3. Backend

3.1 My Reports — reuse analytics.py

No changes. Frontend calls existing whitelisted endpoints:

  • fetch_leaderboard(period, limit)
  • fetch_velocity(project, weeks)
  • fetch_streak(project)
  • fetch_daily_completions()

3.2 Projects + Team — extend portal_reports.py

One new endpoint:

@frappe.whitelist()
def list_managed_projects():
    """Projects where current user is leader/manager.
    Each entry includes KPI snippet: velocity_avg, risk_count, member_count, status.
    """
    _check_flag()
    _require_leader()
    user = frappe.session.user
    return _cache(
        f"managed_projects:{user}",
        lambda: _build_managed_projects(user),
    )

Reuse existing portal_reports endpoints for drill-downs:

  • get_velocity_trend(project) → ProjectDetail velocity section
  • get_forecast(project) → ProjectDetail forecast card
  • evaluate_risks(project) → ProjectDetail risk flags
  • get_okr_rollup(project) → ProjectDetail OKR rollup
  • get_leaderboard(scope='team', period) → TeamReport leaderboard
  • get_overdue(scope='team') → TeamReport overdue list

3.3 Team scope helper

New private helper in portal_reports.py:

def _team_projects(user: str) -> list[str]:
    """Returns project names where user is leader/manager.
    Used to scope team-wide queries to the union of managed-project members.
    """

Existing endpoints accept an optional project_filter=[...] list; if absent, add the parameter and default to all projects.

3.4 Permission + caching

  • _check_flag() + _require_leader() on every project/team endpoint.
  • My Reports endpoints already member-safe.
  • 5-minute TTL cache; key includes user.

4. Page Designs

4.1 Landing (/m/reports)

  • Sticky header Reports.
  • Row cards: 40px square icon + title + sub. Tap → drill.
  • Cards rendered based on useReportsAccess():
  • Member → 1 card (My Reports).
  • Leader with managed projects → 3 cards.
  • Leader with 0 projects → 2 cards (My Reports + My Team if team non-empty, else 1).

4.2 MyReports (/m/reports/me)

  • Reuses current Analytics.tsx content verbatim.
  • Sticky header My Reports + back arrow.
  • Tabs: Leaderboard, Velocity, Streak.

4.3 ProjectsList (/m/reports/projects)

  • Sticky header Projects I Manage + back.
  • Card per project: name, status chip, 3 KPI mini-chips (avg velocity, risk count, member count).
  • Tap → ProjectDetail.
  • Empty: "No projects to report on."

4.4 ProjectDetail (/m/reports/projects/:id)

  • Sticky header: project name + back.
  • Period chips (Week/Month/Quarter) — URL ?period=.
  • Sections (vertical scroll):
  • VelocityChart.
  • Forecast card (target, projected, gap).
  • Risk flags list (color-tagged).
  • OKR rollup table (compact mobile layout).

4.5 TeamReport (/m/reports/team)

  • Sticky header My Team + back.
  • Period chips.
  • Sections:
  • LeaderboardTable (top 10, team-scoped).
  • Completion ring chart (overall %).
  • Overdue list (count + tap → task).
  • Workload bar chart (per member).

5. Data Flow + State

  • React Query keys:
  • ["reports","managed-projects"]
  • ["reports","project", id, period]
  • ["reports","team", period]
  • Existing: ["leaderboard"], ["velocity", p], ["streak", p], ["daily-completions"].
  • staleTime: 60_000 (matches existing analytics).
  • Period chip state via URL search param (?period=month).
  • Project selection lives in URL path, not React state.

Telemetry events:

  • reports_landing_view
  • reports_card_tap { card }
  • reports_my_view
  • reports_project_view { project }
  • reports_team_view
  • reports_period_change { scope, period }

6. Error + Loading

  • Each section: Skeleton while loading, EmptyState w/ Retry on error.
  • 403 from backend → toast + navigate back to /m/reports.
  • Feature flag off → ComingSoon component.
  • Offline → existing OfflineBanner; React Query serves stale cache.
