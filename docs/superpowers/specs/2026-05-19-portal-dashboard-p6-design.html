1. Scope

Redesign halaman /portal (Dashboard) dari placeholder card "Coming soon" menjadi dashboard fungsional yang:

  • Menampilkan konten berdasarkan role aktif user di masing-masing project
  • Mendukung satu user dengan multi-role lintas project (Owner di satu project, Leader di project lain, Member di project lain)
  • Menerapkan top-down workflow: Leader context → Owner context → Member context (urutan default, bisa di-reorder)
  • Responsive: full sidebar (≥1024px) → icon rail (768–1023px) → bottom nav (<768px)
  • Light theme dengan glassmorphism dan purple/indigo accent

Out of scope (P6):

  • Role rename di Frappe backend (VT Manager → Project Owner dll) — dilakukan terpisah
  • Drag-reorder persistensi ke server — disimpan di localStorage saja untuk P6
  • Gamification/points widget — sudah ada di mobile PWA, tidak duplikasi di portal

2. Role Model

Tiga role Frappe yang sudah ada, dengan rename label (backend tetap sama):

Frappe RoleLabel PortalSingkatanWarna
VT Manager / System ManagerProject OwnerOwnerIndigo/Purple
VT LeaderProject LeaderLeaderSky/Blue
VT Member / EmployeeProject MemberMemberGreen

Satu user bisa memiliki lebih dari satu role sekaligus (lintas project yang berbeda). Dashboard menampilkan semua context yang relevan secara bersamaan dalam section terpisah.

4. Components

4.1 Summary Bar

5 metric card horizontal, full width. Data diambil dari satu API call aggregated.

MetricSourceColor
Team BlockedTasks dengan status blocked di project yang user leadRed
Unassigned TasksTasks tanpa assigned_to di sprint aktif yang user leadAmber
OKR OrgRata-rata progress semua OKR aktifGradient purple
Overdue SayaTasks assigned_to=me dengan deadline < todayRed
Sprint — hari tersisaSprint aktif terdekat untuk userAmber

Jika user tidak memiliki role Leader: "Team Blocked" dan "Unassigned Tasks" disembunyikan, diganti metric relevan (poin, rank).

4.2 Section: As Project Leader

Hanya ditampilkan jika user memiliki role VT Leader atau VT Manager di minimal satu project.

Team Pulse:

  • Grid 3 kolom member card
  • Tiap card: avatar, nama, task aktif, PDCA phase badge, status (on track / blocked / idle)
  • Card blocked: border merah, tombol "Bantu" (link ke task detail)
  • Card waiting review: tombol "Review" (link ke task)
  • Data dari portal_tasks.get_team_pulse(project_id) — endpoint baru

Unassigned Tasks:

  • List tasks di sprint aktif tanpa assigned_to
  • Tiap row: PDCA badge, nama task, sprint label, tombol "Assign"
  • Tombol "Assign" buka modal assign (re-use komponen existing)
  • Data dari portal_tasks.get_unassigned_tasks(project_id)

Project selector: Jika user lead lebih dari satu project, tampil dropdown di header section untuk ganti project.

4.3 Section: As Project Owner

Hanya ditampilkan jika user memiliki role VT Manager atau System Manager.

OKR Progress + Trend:

  • List semua OKR aktif: nama, persentase, progress bar, trend arrow (↑↓→) vs minggu lalu
  • OKR di bawah threshold (<30% dengan deadline dalam 2 minggu) diberi border merah + label "🚨 At Risk"
  • Data dari endpoint okr.get_okr_summary() yang sudah ada, tambah field trend_delta

Project Portfolio:

  • List semua project yang user steer: nama, sprint aktif, sisa hari, progress bar, RAG status dot
  • RAG: Green (≥70%), Amber (40–69%), Red (<40%)
  • Data dari projects.get_portfolio_summary() — endpoint baru

4.4 Section: As Project Member

Selalu ditampilkan untuk semua user (setiap user adalah member di minimal satu project).

My Tasks:

  • List tasks assigned_to=me, diurutkan: overdue dulu → due today → upcoming
  • Maksimal 5 ditampilkan, "+N lainnya" link ke full task list
  • Tiap row: checkbox (toggle done via existing API), nama task, PDCA badge
  • Row overdue: border merah subtle
  • Data dari portal_tasks.get_my_tasks() yang sudah ada

Timeline 7 Hari:

  • Horizontal scroll, 7 kolom: H-3, H-2, H-1, Hari Ini, H+1, H+2, H+3
  • Tiap kolom: label hari, tanggal, task chip (nama task + PDCA badge)
  • Warna: H-3 s/d H-1 merah (overdue), Hari Ini ungu highlight, H+1 dst abu-abu normal
  • Data dari portal_tasks.get_my_tasks_timeline(days_back=3, days_forward=3) — endpoint baru

4.5 Drag-Reorder Sections

  • Setiap section punya drag handle (⠿) di kiri header
  • Implementasi: HTML5 Drag and Drop API (sudah di mockup, tidak perlu library)
  • Urutan disimpan ke localStorage key vt_dashboard_section_order sebagai array ['leader','owner','member']
  • Default order: ['leader', 'owner', 'member']
  • Section yang tidak applicable (user tidak punya role itu) tidak ditampilkan sama sekali

4.6 Collapse/Expand Sections

  • Klik header section → toggle collapse
  • State disimpan ke localStorage key vt_dashboard_collapsed sebagai object {leader: false, owner: true, member: false}
  • Badge di header tetap visible saat collapsed (blocked count, unassigned count, dll)

7. Data Flow

DashboardPage mount
  → usePermissions() → determine visible sections
  → useSectionOrder() → read localStorage, apply default
  → useSectionCollapse() → read localStorage
  → render SummaryBar (useDashboardSummary)
  → render sections in order:
      LeaderSection (if hasRole VT Leader or VT Manager)
        → useTeamPulse(projectId)
        → useUnassignedTasks(projectId)
      OwnerSection (if hasRole VT Manager or System Manager)
        → useOkrSummary() [existing]
        → usePortfolioSummary()
      MemberSection (always)
        → useMyTasks() [existing]
        → useMyTasksTimeline()

8. Styling

  • Theme: Light — background #f4f3ff, surface #ffffff, border subtle
  • Glass cards: background: rgba(255,255,255,0.72), backdrop-filter: blur(20px), shadow 0 1px 3px rgba(0,0,0,0.06), 0 4px 16px rgba(124,58,237,0.06)
  • Accent: Indigo #6366f1 / Purple #7c3aed / Violet #8b5cf6
  • Leader color: Blue #2563eb (strip + badges)
  • Owner color: Purple #7c3aed (strip + badges)
  • Member color: Green #16a34a (strip + badges)
  • CSS variables mengikuti pola yang sudah ada di pwa/src/portal/

9. Testing

Unit tests (Vitest + happy-dom):

  • SummaryBar: renders correct metric count, hides leader metrics if no leader role
  • LeaderSection: hidden jika tidak ada role leader, shows blocked count, shows unassigned list
  • OwnerSection: hidden jika tidak ada role owner, renders OKR trend arrows
  • MemberSection: always visible, renders task list + timeline
  • useSectionOrder: returns default order, persists to localStorage, reorders on drag
  • TaskTimeline: maps tasks ke hari yang benar, colors overdue red

Integration tests (Python):

  • test_portal_dashboard.py: 3 skenario — Manager (semua section visible), Leader (no Owner section), Member-only (only Member section)
  • Test get_summary() returns correct counts per role
  • Test get_team_pulse() returns blocked flag correctly
  • Test get_my_tasks_timeline() groups by deadline date

10. Feature Flag

Gate: VT Settings.portal_dashboard_v2_enabled (Check, default 0).
Jika off: tampilkan Dashboard placeholder existing.
Jika on: tampilkan DashboardPage baru.
Flag independent dari flag portal lain.

11. Open Questions

  • Q1: Role rename (VT Manager → Project Owner) — apakah dilakukan di P6 atau terpisah? Saat ini spec ini menggunakan label display saja, Frappe role tetap.
  • Q2: Jika user lead >1 project di Leader section — apakah aggregate semua project atau pilih satu via dropdown? Spec ini default aggregate, dropdown sebagai enhancement.
  • Q3: Timeline: apakah tasks dari semua project atau hanya project aktif? Default: semua project di mana user adalah member.
