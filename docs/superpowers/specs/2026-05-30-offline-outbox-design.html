Offline Outbox — Sync Queue + Retry

Date: 2026-05-30
Scope: v2 — let task mutations be queued while offline (no optimistic UI), drained on reconnect or via a "Sync now" button. Upgrade OfflineBanner to show pending count.
Approach (approved): Outbox WITHOUT optimistic updates — queued actions show as "pending"; UI does not pretend the mutation already applied.
Pre-state (verified): No outbox exists. Mutations are explicitly blocked offline (button disabled + toast "Sambungkan internet dulu"). Read-cache in IndexedDB via idb-keyval; sync timestamps in localStorage. offline_seen telemetry defined but never fired.

1. Architecture

FileResponsibility
cache/outbox.tsIndexedDB-backed queue of pending mutations (enqueue, list, remove, count) keyed under vt:outbox:.
hooks/useOnline.tsShared online/offline state (replaces ad-hoc navigator.onLine checks).
sync/outboxRunner.tsDrains the outbox: dispatches each queued action to its API, removes on success, retains + backs off on failure. Single-flight (no concurrent drains).
hooks/useOutbox.tsReact binding: exposes pendingCount, syncing, syncNow(); auto-drains on reconnect.
components/OfflineBanner.tsxUpgraded: shows offline state AND pending count + "Sync now" button (visible whenever pending > 0, even online).

2. Outbox model

An outbox entry is a serializable description of one mutation:

interface OutboxEntry {
  id: string;            // uuid (crypto.randomUUID)
  kind: "complete" | "log_progress" | "snooze" | "create_task";
  payload: Record<string, unknown>;  // args for the API call
  createdAt: number;     // epoch ms
  attempts: number;      // retry counter
  lastError?: string;
}
  • Stored as a single array under one idb key (vt:outbox:queue) for simplicity; small N expected.
  • kind maps to an API dispatcher in outboxRunner (a registry: kind → function(payload)). New mutation kinds register here.
  • No optimistic cache writes. After a successful drain, the runner invalidates the affected react-query keys so the UI refetches the true server state.

3. Enqueue flow (mutation interception)

Today the mobile actions check navigator.onLine and block when offline. Change that branch: when offline, instead of the toast-and-abort, enqueue the action and show a toast "Disimpan, akan dikirim saat online" + the pending badge updates. When online, behave exactly as today (direct mutation).

  • Sites to update: MyWork/List.tsx complete/log/snooze; QuickAddTaskModal create (create_task). Each already has the params needed for the API.
  • The undo-window (useUndoableMutation) stays for online completes; offline path skips the undo window and enqueues directly.
  • Keep mutations that are not safe to queue (e.g. comment add) online-only for now — out of scope.

4. Drain + retry

  • Trigger: (a) automatically when online transitions false→true; (b) manually via "Sync now".
  • Order: FIFO by createdAt.
  • Per entry: call dispatcher; on success remove entry + collect affected query keys; on failure increment attempts, store lastError, keep entry, stop the run (do not block the queue head forever — but a poison entry exceeding MAX_ATTEMPTS (5) is moved to a "failed" state surfaced in the banner as "N gagal" with a retry/discard affordance).
  • Single-flight: a module-level flag prevents overlapping drains.
  • After run: invalidate collected query keys once; stamp last-sync time.

5. Banner UX

  • Offline + no pending → current message ("Mode offline · terakhir sinkron …"), danger background.
  • Pending > 0 (offline OR online) → "N aksi menunggu" + a "Sync now" button (disabled while syncing; shows spinner text "Menyinkronkan…").
  • Failed > 0 → "N gagal" with "Coba lagi" (re-queues) — keep minimal.
  • Online + zero pending → banner returns null (unchanged).
  • a11y: banner stays role="status"; Sync button is a real <button> with accessible label.

6. Telemetry

Fire (add to TelemetryEvent union): outbox_enqueue ({kind}), outbox_drain_start ({count}), outbox_drain_done ({ok, failed}), offline_seen (wire the existing-but-unused event when banner first shows offline).

7. Testing

  • outbox.ts: enqueue/list/remove/count round-trip against a fake idb (mock idb-keyval like cache.test.ts does).
  • outboxRunner.ts: drains in FIFO; removes on success; retains + increments attempts on failure; single-flight; moves to failed after MAX_ATTEMPTS.
  • useOnline: reflects online/offline events.
  • OfflineBanner: renders pending count + Sync button when pending>0; Sync click calls syncNow; null when online & empty.
  • Enqueue integration: offline complete in MyWork enqueues instead of toast-abort (mock useOnline=false).
  • Gate: full pnpm vitest run + tsc + build.

8. Out of scope

  • Optimistic UI / local echo of queued mutations.
  • Conflict resolution beyond last-write-wins on the server.
  • Service-worker background-sync (Workbox plugin) — app-level drain only for now.
  • Queuing comment/non-task mutations.
