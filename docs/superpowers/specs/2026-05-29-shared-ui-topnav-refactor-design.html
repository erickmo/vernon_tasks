Refactor: Shared Badge/Modal + TopNav Decompose

Date: 2026-05-29
Scope: v2 tech-debt refactor — extract shared Badge and Modal primitives + a useDismiss hook, decompose the 629-line TopNav.tsx. No behavior change for end users; adds a11y (role/aria/focus-trap/esc) to modals.
Pre-state (verified): 5 badge implementations across 4 files; 10 modal/overlay implementations (none with role/aria/focus-trap); TopNav.tsx = 629 lines with inline NotificationDropdown + AvatarDropdown + icons + 16 hardcoded color constants duplicating theme vars.

1. New shared primitives

FileResponsibility
components/ui/Badge.tsxUnified count/dot badge
hooks/useDismiss.tsEscape key + outside-click + focus-trap (used by Modal AND the NotificationBell popover)
components/ui/Modal.tsxDialog wrapper with center / sheet / slide variants
components/TopNav/Folder: TopNav.tsx shell + extracted parts

2. Badge

Props: { count?: number; max?: number /* default 99 */; variant: "dot" | "count"; tone?: "danger" | "primary" /* default danger */; ring?: boolean; ariaLabel?: string }.

  • variant="dot" → 8px circle, no text (BottomNav unread indicator).
  • variant="count" → 16px min-width rounded pill, shows count, caps at {max}+ (e.g. "99+"). Optional ring (2px ring in nav gradient context).
  • Colors from tokens: var(--vt-danger) / var(--vt-primary). No hardcoded hex (replaces #ef4444, #f87171).
  • Returns null when variant==="count" and count falsy/0 (matches current "hide when 0" behavior).

Migrate (5): BottomNav.tsx dot, TopNav notification count (ring), portal TopBar.tsx NavBadge (CSS class → component; remove .portal-topbar__nav-badge rule from portal.css), NotificationBell.tsx badge, Me.tsx unread.

3. Modal

Props: { open: boolean; onClose: () => void; variant: "center" | "sheet" | "slide"; labelledBy?: string; busy?: boolean; children: ReactNode }.

  • Renders an overlay + a container styled per variant: center = translate(-50%,-50%); sheet = bottom sheet with border-radius:16px 16px 0 0 + --safe-bottom padding; slide = right panel width 360.
  • A11y: container gets role="dialog", aria-modal="true", aria-labelledby={labelledBy} when provided. Focus trap on open (focus first focusable, cycle Tab, restore focus on close). Escape calls onClose. Backdrop click calls onClose unless busy is true.
  • Overlay opacity + z-index preserved per current usage (center 60/61, sheet 100, slide 50/51). Modal accepts an optional zIndex override (default by variant) so existing stacking is kept.
  • Body content (forms, fields) stays in each modal; only the shell/overlay/a11y plumbing moves into Modal.

Migrate (8 true modals): ProjectFormModal, QuickAddTaskModal (center); FilterSheet, RejectModal, LogProgressModal (sheet); ReloginModal, IOSInstallModal (center); TaskSlideOver (slide). TaskCreateModal (portal) already has role=dialog aria-modal — migrate its overlay/dismiss to Modal keeping its markup.

Popover exception (approved): NotificationBell is a popover (button-anchored computed position, no backdrop) — NOT forced into Modal. It instead adopts useDismiss for esc/outside-click and gets a role. Its inner NotificationPanel already has role="dialog".

4. TopNav decompose

Split components/TopNav.tsx into a components/TopNav/ folder:

  • TopNav.tsx — shell: layout, breadcrumb row, nav2 items, composes the parts.
  • NotificationDropdown.tsx — extracted (props { unread }, its query/state). Uses shared Badge + useDismiss.
  • AvatarDropdown.tsx — extracted (props { username }). Uses useDismiss.
  • icons.tsx — IconBell, IconChevron, LogoMark.
  • breadcrumb.ts — BREADCRUMB_MAP, NAV2_ITEMS, getBreadcrumb, getInitials.

Tokens: replace C_BG/C_SURFACE/C_BORDER/C_TEXT/C_MUTED/C_PRIMARY/C_PRIMARY_L/C_DANGER/C_DANGER_L with existing --vt-* tokens. The nav-gradient set (C_NAV_BG/C_NAV_TEXT/C_NAV_MUTED/C_NAV_BORDER/C_NAV_ACTIVE) has no token equivalent → add --vt-nav-bg, --vt-nav-text, --vt-nav-muted, --vt-nav-border, --vt-nav-active to theme/tokens.css and reference them. Import path for TopNav must stay resolvable (folder index or same export name) so existing importers don't change.

5. Testing

  • New unit tests: Badge (dot renders 8px; count renders number; caps at {max}+; returns null on 0). Modal (renders children when open; role="dialog" + aria-modal; Escape → onClose; backdrop click → onClose; backdrop click ignored when busy; focus moves into dialog on open). useDismiss (esc + outside-click fire handler).
  • Regression: every migrated modal/badge keeps its EXISTING test file green (behavior unchanged). TopNav.test.tsx must stay green after the split.
  • Gate: full pnpm vitest run + tsc --noEmit + pnpm build after the work.

6. Sequencing (risk-controlled)

  1. Build primitives first with tests: useDismiss → Badge → Modal.
  2. Add --vt-nav-* tokens.
  3. Migrate badges (5) — small, low risk.
  4. Migrate modals one-by-one (center → sheet → slide → portal), running the modal's test after each.
  5. Decompose TopNav last (depends on Badge + useDismiss), keeping TopNav.test.tsx green.
  6. Final full-suite + lint + build gate.

7. Out of scope

  • Visual redesign of any modal/badge — pixels stay the same (only structure + a11y change).
  • Dark mode (separate v2 item).
  • Sprint kanban mobile, offline sync queue (separate v2 cycles).
