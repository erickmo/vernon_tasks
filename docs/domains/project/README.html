Validations & Lifecycle Rules

All rules in doctype controllers per ADR-022. Child rows are validated explicitly via parent _validate_children wiring (Frappe doesn't auto-call child validate()).

VT Project

HookRule
validatetitle normalize + ≤ 200 char; end_date strictly > start_date
validatePDCA transition guard (PLAN→DO→CHECK→ACT/CLOSED, ACT→PLAN/DO; CLOSED terminal); pdca=CLOSED forces status=Closed
validatelinked Objective brand must equal project.brand
validateowner/leader cannot appear in team_members
on_trashblock delete while VT Task or VT Sprint links exist

VT Sprint

Rule
sprint_title normalize + ≤ 140 char; end_date > start_date; range fits inside project.start..end
Status flow forward-only: Planning → Active → Review → Closed
Every Sprint Task in tasks must belong to the same project; no duplicate task rows

Sprint Task

Rule
task required; must reference an existing VT Task

Project Team Member

Rule
user required + must exist; role ∈ {Owner, Leader, Member}
is_also_leader rejected on Owner rows (Owner already outranks)

Project Milestone

Rule
milestone_title normalize + ≤ 200 char; status ∈ {Open, Completed}
due_date (when set) must be inside parent project's date range

Project Documentation

Rule
doc_title normalize + ≤ 200 char; content ≤ 50000 char
