  • vt_contact_request — landing-page contact form submissions
  • vernon_telemetry_event — append-only product analytics

Validations & Lifecycle Rules

All rules in doctype controllers per ADR-022.

VT Settings (Single)

Rule
weight_multiplier ∈ (0, 1000]
early_bonus_rate / late_penalty_rate / revision_deduct_rate ∈ [0, 1.0] (fractions per day)
default_daily_target_hours ∈ (0, 24]
default_blocked_days_threshold ≥ 0

Vernon Push Subscription

Rule
user must exist; endpoint required + must be https://
p256dh + auth required (JSON length caps applied)
endpoint immutable after insert (re-subscribe = delete + new row)
before_insert stamps last_seen if blank

Vernon Push Preference

Rule
user must exist; user is the permanent PK
before_rename blocked

VT Contact Request

Rule
full_name normalize + ≤ 200 char; company ≤ 200; message ≤ 5000
email server-side format validation (validate_email_address); lowercased canonical
status ∈ {New, Contacted, Closed}

Vernon Telemetry Event

Rule
event name matches [a-zA-Z0-9_.]{1,64} (group-by friendly)
timestamp not in future (5-min clock-skew tolerance, site TZ aware)
props ≤ 10000 char + must parse as JSON when set
before_insert defaults: timestamp = now_datetime(), user = session.user
