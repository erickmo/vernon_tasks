Validations & Lifecycle Rules

All rules in doctype controllers per ADR-022. Frappe Hooks-First.

Work Profile

Rule
daily_target_hours ∈ (0, 24]
work_start_time < work_end_time when both set
working_days: child validate wired; no duplicate day_of_week

Work Schedule Day (child)

Rule
day_of_week required + must be a weekday name (Mon..Sun)
When is_working & both times set, start_time < end_time

Daily Summary

Rule
user must exist; date required, ≤ today (no future)
target / scheduled / completed hours each ∈ [0, 24]
total_points_today is signed (negative allowed for net penalty days)
Natural unique key (user, date) via name format DS-{user}-{date}

User Point Summary

Rule
user must exist; period grammar matches Objective (YYYY | YYYY-Hn | YYYY-Qn | YYYY-MM)
total_earned / total_bonus / total_penalty ≥ 0 (raw aggregates)
total_override_delta signed (overrides can subtract)
net_points auto-synced on save: earned + bonus − penalty + override_delta
Natural unique key (user, period) via name format UPS-{user}-{period}
