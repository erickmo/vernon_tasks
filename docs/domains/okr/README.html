Domain: OKR

Objectives, Key Results, KPI definitions and entries. Scoped by VT Brand. Feeds Health Score.

Hierarchy

VT Brand
  └── Objective (period, owner, pdca, brand reqd)
        ├── Key Result (target/current/progress)
        └── KPI Definition (brand fetched from objective) → KPI Entry

VT Project also links to Brand. If a Project links an Objective, project.brand must match objective.brand (enforced in vt_project.validate).

Modules

  • vernon_tasks/okr/doctype/ — objective, key_result, kpi_definition, kpi_entry
  • Report: kpi_achievement

Health Score

score = okr_pct × 0.50 + ontime_pct × 0.30 + velocity_health × 0.20

Service: vernon_tasks/task/services/health_score_service.py + test_health_score_service.py.

Two entry points:

  • get_health_score(brand=None) — global if no brand, scoped if brand passed. SQL filters Objective.brand, VT Task→Project.brand, VT Project.brand.
  • list_brand_health_scores() — iterates VT Brand, returns one snapshot per brand. Exposed via exec_analytics.list_brand_health.
