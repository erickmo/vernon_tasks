Domain: Brand

A VT Brand is the top-level scope unit. Projects, Objectives, KPI Definitions, and Health Score snapshots are all scoped by Brand. Brand is also the unit of multi-tenant separation in the dashboard rollup.

Modules

  • vernon_tasks/brand/doctype/vt_brand/ — controller, JSON schema, tests
  • vernon_tasks/brand/api/brand_strategy.py — whitelisted endpoints (delegates to controller)
  • vernon_tasks/brand/api/portal_brands.py — whitelisted desk API (list/search/get/create/update/delete + permissions)
  • vernon_tasks/task/page/vt_brands/ — desk Brand management page (card grid, route /app/vt-brands)

Validations & Lifecycle Rules

All rules live in vt_brand.py (Frappe controller hooks) so the standard REST endpoints (/api/resource/VT Brand) enforce them automatically — no custom wrapper needed. See ADR-022 — REST-first, hooks-for-logic.

HookRuleFailure
autonameReject control chars in raw input; collapse whitespace; trimValidationError — "Nama brand tidak boleh mengandung karakter kontrol"

Avatar Generation

When no logo is uploaded, the controller renders a deterministic initial-based SVG avatar:

  • First non-whitespace char of brand_name uppercased
  • Background colour picked from a 10-colour palette using md5(brand_name)[0] % 10 (stable across renders)
  • Persisted as a public File doc (filename: brand-avatar-{slug}.svg) — not a data URL, because Attach Image is VARCHAR(140) and a data URL would overflow
  • User-uploaded logos (any path NOT starting with /files/brand-avatar-) are never overwritten

Desk Page: vt-brands

Desk Page vt-brands (route /app/vt-brands, module Task, title Brand) menampilkan VT Brand sebagai grid kartu yang dapat diklik di Frappe Desk. Halaman ini bersifat list/read-only: semua aksi create/edit/delete didelegasikan ke form VT Brand native — tidak ada UI edit/hapus di halaman itu sendiri.

  • Kartu, bukan tabel — grid flex .vh-row berisi .vh-card (menggunakan ulang styling vt-home/vh-*). Empty state: Belum ada brand.
  • Klik kartu membuka form native: frappe.set_route('Form', 'VT Brand', b.id).
  • Field per kartu: brand_name (judul tebal, di-escape), logo (img 40×40, border-radius 8, object-fit cover; fallback ke badge inisial indigo #6366f1 dari brand_name[0] bila kosong), dan description (di .vh-item-meta, dipotong pada DESC_MAX = 80 char + elipsis). Semua teks di-escape via frappe.utils.escape_html (XSS-safe).
  • Sumber data: vernon_tasks.brand.api.portal_brands.list_brands (konstanta BRANDS_API), dipanggil saat load dan tombol Refresh. Halaman memanggil tanpa argumen search (filter search tidak diwire di UI). list_brands mengembalikan {id, brand_name, logo, description} (id = doc.name), butuh login + frappe.has_permission('VT Brand','read'), urut brand_name ASC, limit 500.
  • Gate izin: tombol primary Buat Brand (frappe.new_doc('VT Brand')) hanya diset bila frappe.model.can_create('VT Brand') true (helper client, didukung frappe.boot.user.can_create; frappe.has_permission hanya server-side). Akses halaman (vt_brands.json roles): VT Member, VT Leader, VT Manager — semua bisa membuka halaman; tombol buat tetap bergantung pada cek can_create per-user.
  • Konvensi script: seluruh JS dibungkus IIFE ((function () { ... })();) agar const/let top-level tidak bocor ke scope global desk dan bentrok (Identifier X has already been declared) saat re-eval / pindah antar halaman VT. Background abu diterapkan global oleh vt_page_style.js.

Cross-References

  • VT Project (project domain) — link field brand. Enforced FK from this side via on_trash.
  • Objective (okr domain) — link field brand (required). If a Project links an Objective, project.brand must equal objective.brand (enforced in vt_project.validate).
  • KPI Definition (okr) — brand fetched from parent Objective.
  • Health Score Service (task/services/health_score_service.py) — list_brand_health_scores() iterates VT Brand to produce per-brand snapshots.
