ADR-007 — Strict layer separation: Controller / Service / Page

Status: Accepted
Context

Frappe doctype controllers + Page handlers tend to absorb logic → untestable, duplicated, fragile. Vernon adds ~20 doctypes; risk of God classes.

Decision
  • Controller (doctype/<name>.py): validation, db_set, lifecycle hooks. No math.
  • Service (task/services/*.py): logic, calculations, algorithms. Pure when possible. DI for settings.
  • Page handler (page/<name>/<name>.py): auth check, request parse, call services, format response.

Function ≤ 40 lines. No class > 300 lines / 5 responsibilities. Named constants only.

Consequences
  • + Services unit-testable without Frappe boot.
  • + Easy onboarding.
  • − More files; navigation discipline required.

ADR-008 — PDCA as first-class state machine on Task & Project

Status: Accepted
Context

Free-form status fields drift; can't enforce review gates. Gamified points only fair if every Task funnels through review.

Decision

Define pdca_phase field with strict transition matrix per doctype. Validator rejects illegal transitions. Kanban Status mapped 1:1 from PDCA Phase.

VT Task: BACKLOG → PLAN → DO → CHECK → DONE
                          ↑     │
                          └─ ACT┘

Internal services use db_set("pdca_phase", phase, update_modified=False) to bypass validation only in trusted code paths.

Consequences
  • + Audit-friendly: every state change traceable.
  • + Leader review = enforced bottleneck for points.
  • − More transitions to enumerate.

ADR-009 — Immutable Task Point Log for all point math

Status: Accepted
Context

Points feed compensation discussions / leaderboards. Computing from current task fields = no audit; manager overrides leave no trail.

Decision

Every point delta writes append-only row to Task Point Log: task, type (earned/early_bonus/late_penalty/revision_deduction/leader_override), amount, by_user, reason, timestamp. Monthly aggregate cached to User Point Summary.

Consequences
  • + Full audit. Overrides visible to stakeholders.
  • + Reconstructable totals if cache lost.
  • − Storage grows linearly; mitigated by indexed reads + monthly rollup.
