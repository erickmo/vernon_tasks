ADR-010 — Web Push (VAPID) over native shells (Capacitor)

Status: Accepted
Context

Need push for review queue / assignment / due reminders. iOS Web Push only on PWA installed via A2HS (iOS 16.4+). Native shell = build pipeline, app store distribution.

Decision

Web Push API with VAPID keys generated once via bench vernon-generate-vapid → stored in VT Settings. Backend doctype Vernon Push Subscription. Notification Log.after_insert hook dispatches push. Dead endpoints (404/410) auto-pruned.

Consequences
  • + Zero app-store overhead. One install path: A2HS.
  • − iOS users must install PWA (≥16.4); modal hints user.
  • − No background sync APIs on iOS (acceptable).

Re-evaluate if iOS adoption blocked → future ADR for Capacitor.

ADR-011 — Recharts for charts, lazy-loaded

Status: Accepted
Context

Burndown, velocity, KPI trend, leaderboard need responsive interactive charts. Recharts is React-idiomatic but heavy (~357 KB raw).

Decision

Use Recharts. Analytics + Leader pages lazy-loaded via React.lazy. Recharts shared chunk loaded only when entering chart pages. Main bundle pays nothing.

Consequences
  • + Main bundle stays ~95 KB gz.
  • − First chart-page visit = larger payload; cached after.
Alternatives
  • Chart.js: imperative, harder to compose with React.
  • visx/d3 from scratch: too much custom code.

ADR-015 — Optimistic UI w/ 5s undo; no offline mutation queue

Status: Accepted
Context

"Complete task" must feel instant on mobile. Real offline write-queue needs conflict resolution + sync layer.

Decision

useUndoableMutation shows toast w/ Batalkan button; setTimeout 5 s, then sends real request. If offline → reject up-front, toast "Sambungkan internet dulu". No queued retries.

Consequences
  • + Sub-100 ms perceived latency on completes.
  • + Simple mental model; no conflict edge cases.
  • − No "do it offline, sync later" UX (acceptable for current users).

ADR-016 — One PWA codebase serves mobile + desktop (breakpoint 768px)

Status: Accepted Date: 2026-05-14
Context

Leaders + execs working from laptops want same data as mobile PWA. Separate desktop UI = duplicate effort.

Decision

Single React codebase. useMediaQuery(768) drives layout. <768 px = BottomNav (5 tabs). ≥768 px = TopNav two-tier (Nav1 + Nav2 submenu). Page layouts adapt: Dashboard 2-col grid, Work master-detail ≥900 px. Tab state via useSearchParams for deep-link.

Consequences
  • + Single shipped artifact; one test matrix + viewport.
  • + Desktop users get install + offline + push for free.
  • − Layout complexity per page; mitigated by composable layout primitives.

ADR-017 — id-ID formal locale ("Anda") as baseline

Status: Accepted
Context

Vernon Corp is Indonesia-based; primary user language = Bahasa Indonesia. Mixed en/id UI creates cognitive load.

Decision

All user-facing strings = id-ID formal ("Anda", not "kamu"). Dates DD MMM YYYY (16 Mei 2026), times HH:mm. Strings centralized in i18n.ts. English left only in code identifiers + dev docs.

Consequences
  • + Consistent professional tone; lower friction.
  • − Multi-locale support deferred (no urgent need).
