1. Executive Summary

Vernon Tasks = company-wide task / project / performance platform on Frappe v15. Unifies three methodologies under one doctype graph:

  1. PDCA Cycle — Plan-Do-Check-Act lifecycle on every Task, Project, Objective.
  2. OKR / KPI — Strategic objectives + measurable Key Results + KPI definitions, rolled up to company Health Score.
  3. Agile Sprints — Time-boxed iterations with velocity, burndown, forecast analytics.

Gamified point system rewards on-time delivery, penalizes late completion / revision cycles. Points feed monthly leaderboards, personal streaks, team velocity.

Ships two coordinated UIs (historical — the product is now desk-only; see removal notice above):

  • Frappe Desk (desktop, Python/Jinja) — custom pages for members, leaders, execs (now the sole UI; post-login landing /app/vt-home).
  • Vernon PWA (mobile-first React) — installable, offline-capable, push-enabled, at /m/. (removed — desk-only migration)

Vision

Make execution status, blockers, outcomes visible to every member in real time, on any device — reward consistent delivery with auditable, math-driven points.

Non-Goals

  • External-facing customer portal.
  • Replacement for Frappe HR / ERPNext Projects / generic CRM.
  • Time-tracking for billing (hours = capacity/effort only).

4.4 Point Calculation

base               = weight × weight_multiplier
early_bonus        = base × early_bonus_rate × max(0, days_early)
late_penalty       = base × late_penalty_rate × max(0, days_late)
revision_deduction = base × revision_deduct_rate × revision_count
earned             = base + early_bonus − late_penalty − revision_deduction

Defaults (in VT Settings): weight_multiplier=10, early_bonus_rate=0.05, late_penalty_rate=0.08, revision_deduct_rate=0.10.

Every transaction writes row to Task Point Log (types: earned, early_bonus, late_penalty, revision_deduction, leader_override). Monthly aggregate → User Point Summary.

4.5 Scheduling & Capacity

  • Distribute estimated_hours evenly across assignee's working days (from Work Profile).
  • Manual daily overrides auto-rebalance remainder.
  • Capacity conflict = scheduled hours > daily target; surfaced to leaders.
  • Recurring tasks generated daily by scheduler from Recurring Rule.

9.2 Coding Standards

  • Layer separation: Controller (validation, db_set) · Service (logic, math) · Page Handler (auth + IO).
  • Functions ≤ 40 lines; no God classes > 300 lines / 5 responsibilities.
  • Named constants only — no magic strings/numbers.
  • ignore_permissions=True only inside internal services / scheduled jobs.
  • All whitelisted endpoints call frappe.has_role() / frappe.only_for() at top.
  • Email sends wrapped in try/except (silent skip if mail unconfigured).
  • DI for services — settings/config as params, never instantiated inline.

9.3 Testing

  • Backend: bench run-tests --app vernon_tasks. Coverage ≥ 80 % on services + security.
  • PWA unit/component: Vitest + happy-dom + @testing-library/react.
  • PWA e2e: Playwright (PWA_BASE_URL, PWA_TEST_USER, PWA_TEST_PASS).
  • TypeScript: tsc --noEmit strict.

11. Rollout Strategy

11.1 PWA Pilot (P0.5) Removed — desk-only

Historical rollout plan. The PWA pilot surface and its /m/ rollout were removed in the desk-only migration; retained for product history.

  1. Pre-launch: build green, iOS A2HS works, Android install works, airplane-mode shows cached + banner, session relogin modal opens, pwa_boot events recorded.
  2. Pilot: 1 team (5–10 users), invite-only.
  3. Monitor: error_boundary < 1 % of page_view; login_failure < 5 % post-success.
  4. Go/No-Go gates:
    • error boundary < 1 %
    • login failure < 5 %
    • install rate ≥ 30 %
    • no open P0 bugs
  5. Company rollout: Desk banner → /m/, email announcement, gradual migration. (removed — desk-only; /m/ no longer exists)

11.2 Feature Phasing Removed — desk-only

Phases P0.5 → P4b sequential + additive. Each phase behind feature flag resolved via /m/_boot. Responsive-nav (current) requires all of P0.5–P3b. (removed — desk-only; the _boot feature-flag layer and /m/* phasing were dropped in the desk-only migration)
