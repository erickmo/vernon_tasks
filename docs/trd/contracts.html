5.2 Retention

  • Vernon Telemetry Event: 90 days. Daily purge via scheduler.
  • Task Point Log: indefinite (audit).
  • Notification Log: Frappe default (configurable in Frappe).

6. API Contracts

6.1 Conventions

  • Base URL: POST /api/method/vernon_tasks.<module>.<file>.<fn>
  • Auth: cookie (sid) or Authorization: token <key>:<secret>
  • CSRF: X-Frappe-CSRF-Token on mutations (browser context)
  • Body: application/x-www-form-urlencoded or JSON
  • Response: { "message": <payload> }
  • Errors: HTTP 400 (ValidationError), 403 (PermissionError), 417 (rate-limit / bound), 500

6.2 Sample contracts

my_work.list

request:  POST /api/method/vernon_tasks.task.api.my_work.list
response: {
  "message": {
    "overdue":  [TaskCard],
    "today":    [TaskCard],
    "upcoming": [TaskCard]
  }
}

TaskCard = {
  id: string, title: string, status: string,
  priority: "Tinggi"|"Sedang"|"Rendah",
  due_date: "YYYY-MM-DD", project: string, points: number
}

my_work_mutations.complete

request:  { task_id: string }
response: { ok: true, idempotent?: boolean, task_id: string }
errors:   403 require_login · 417 rate_limit · 400 invalid state

my_work_mutations.log_progress

request:  { task_id, hours: 0.25..8 (step 0.25), note?: string (≤1000) }
response: { ok: true, actual_hours: number }
errors:   417 rate_limit / clamp · 400 invalid hours

analytics.get_burndown

request:  { sprint: string }
response: {
  labels: ["YYYY-MM-DD", ...],
  ideal:  [number, ...],
  remaining: [number, ...],
  unestimated_count: number
}

exec_analytics.get_health_score

response: {
  score: 0..100,
  okr_pct: number,
  ontime_pct: number,
  velocity_health: number,
  breakdown: { okr_weight: 0.4, ontime_weight: 0.3, velocity_weight: 0.3 }
}

push.subscribe

request:  { endpoint, p256dh, auth, user_agent }
response: { ok: true, renewed: boolean }
errors:   417 rate_limit (5/min) · max_str(endpoint, 2048)
