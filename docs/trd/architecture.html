2.2 Request lifecycle (PWA mutation)

  1. User taps "Selesai" on task card.
  2. useUndoableMutation shows 5s undo toast (no API call yet).
  3. After 5s without cancel вҶ’ React Query onMutate applies optimistic update.
  4. fetch('/api/method/vernon_tasks.task.api.my_work_mutations.complete', {...}) with cookie + X-Frappe-CSRF-Token.
  5. Frappe routes вҶ’ page handler вҶ’ require_login() вҶ’ rate_limit("complete", 30) вҶ’ service.
  6. Service mutates VT Task вҶ’ on_submit triggers point_calculator.calculate_points вҶ’ row in Task Point Log вҶ’ User Point Summary update.
  7. Response returns. onSuccess invalidates ["my-work"] query.
  8. If error вҶ’ onError rollback optimistic state + toast.

4. Data Flow Diagrams

4.1 Task completion вҶ’ points

PWA / Desk вҶ’ my_work_mutations.complete(task_id) в””в”ҖвҶ’ security.require_login + rate_limit("complete", 30) в””в”ҖвҶ’ frappe.get_doc("VT Task", task_id).db_set("pdca_phase", "DONE") в””в”ҖвҶ’ doc_event on_submit в””в”ҖвҶ’ point_calculator.calculate_points(task) в”ңв”ҖвҶ’ base = weight Г— multiplier в”ңв”ҖвҶ’ early_bonus or late_penalty в”ңв”ҖвҶ’ revision_deduction в””в”ҖвҶ’ Task Point Log row(s) inserted в””в”ҖвҶ’ User Point Summary recomputed (or +=) в””в”ҖвҶ’ analytics.invalidate_project_cache(project) в””в”ҖвҶ’ Notification Log inserted (assignee / leader) в””в”ҖвҶ’ after_insert вҶ’ push_sender.send_push_for_notification в””в”ҖвҶ’ filter by Vernon Push Preference в””в”ҖвҶ’ POST to each Vernon Push Subscription endpoint в””в”ҖвҶ’ dead endpoint (404/410) вҶ’ prune row
