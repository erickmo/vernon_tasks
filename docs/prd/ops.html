9.2 Coding Standards

  • Layer separation: Controller (validation, db_set) · Service (logic, math) · Page Handler (auth + IO).
  • Functions ≤ 40 lines; no God classes > 300 lines / 5 responsibilities.
  • Named constants only — no magic strings/numbers.
  • ignore_permissions=True only inside internal services / scheduled jobs.
  • All whitelisted endpoints call frappe.has_role() / frappe.only_for() at top.
  • Email sends wrapped in try/except.
  • DI for services — settings/config as params.

9.3 Testing

  • Backend: bench run-tests --app vernon_tasks. Coverage ≥ 80 %.
  • PWA unit/component: Vitest + happy-dom + @testing-library/react.
  • PWA e2e: Playwright.
  • TypeScript: tsc --noEmit strict.

11. Rollout Strategy

11.1 PWA Pilot (P0.5)

  1. Pre-launch: build green, iOS A2HS works, Android install works, airplane-mode shows cached + banner, session relogin modal opens, pwa_boot events recorded.
  2. Pilot: 1 team (5–10 users), invite-only.
  3. Monitor: error_boundary < 1 % of page_view; login_failure < 5 % post-success.
  4. Go/No-Go gates: error boundary < 1 %, login failure < 5 %, install ≥ 30 %, no P0 bugs.
  5. Company rollout: Desk banner → /m/, email announcement, gradual migration.

11.2 Feature Phasing

Phases P0.5 → P4b sequential + additive. Each phase behind feature flag resolved via /m/_boot. Responsive-nav (current) requires all of P0.5–P3b.

Desktop Portal · Nginx asset symlink

Architecture note: /portal/ and /m/ serve the same built PWA bundle (single SPA, branches by URL prefix inside the React router). The built assets live at vernon_tasks/www/m/assets/ (Vite output target), reached in production via the existing sites/<site>/public/m symlink chain documented in memory note project_frappe_pwa_nginx.

For desktop portal deployments, expose those same assets under /portal/assets/. Pick one of:

  • (a) Primary — mirror the /m/ symlink pattern: create sites/<site>/public/portal as a symlink pointing to the same target as the existing sites/<site>/public/m symlink (i.e. vernon_tasks/www/m/). Nginx then serves /portal/assets/* directly with zero extra config.
  • (b) Alternative — Nginx alias: add a location /portal/assets/ block that aliases the same directory used by /m/assets/. Useful where symlinks under sites/public are undesirable.

Without one of these, Frappe's catch-all website_route_rules intercept asset requests and break the bundle.
