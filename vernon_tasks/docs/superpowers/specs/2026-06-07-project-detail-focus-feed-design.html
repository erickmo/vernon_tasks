Project Detail — "Fokus Feed" redesign

Date: 2026-06-07 Task size: L (new default panel + quick-capture + load lens + batch-assign; 2 backend touches; CSS; tests) App ownership: 100% vernon_tasks — no Frappe core files touched Base branch: master Selection: winner of a 4-candidate, 3-judge UX tournament (weighted rubric)

1. Goal

Optimize vt-project-detail for a project leader's daily usage. Two explicit pains drive the redesign: (a) the #1 daily action — create + triage tasks fast — costs too many clicks/modals today; (b) priorities are hard to spot across 7 tabs. The redesign makes priority the page's information architecture and collapses the 7 tabs into a segmented switch with a new Fokus feed as the default.

4. Behavior

4.1 Priority surface (the core)

Priority IS the IA, with redundant encoding so nothing actionable hides:

  1. Bucket order = ranking: Terlambat › Diblokir › Belum Ditugaskan › Berisiko › Jatuh Tempo Hari Ini › collapsed "Lainnya/terjadwal". A task lands in exactly one bucket (first match) so counts never double-add; its row still renders all its flag chips.
  2. Trouble-proportional render: non-empty action buckets auto-expand; empty ones collapse to a muted "0". A healthy project is mostly collapsed greens; a troubled one is a wall of red. The visual mass IS the signal.
  3. Global read: hero risk chip + "⛔N blocker" + toolbar "⚠N perlu aksi" (dedup union) give the total before any scroll.
  4. Card color: priority chips reuse existing .vb-prio-* tokens; rows keep server sort (Critical-first, earliest deadline).
  5. Date-correct overdue: Terlambat/Due-today use the new server overdue:bool (getdate(today()), site-tz) — NOT browser-local Date.
  6. Unassigned self-announces: the 👤 chip renders as an empty "[ pilih ▾]" — a to-do that is also its own fix.

4.2 Create + triage flow

  • Create (≤2 clicks, 0 modal): quick-capture title autofocused on load (n re-focuses) → type → Enter → create_task into the first PDCA column. To capture fully-triaged: pick priority/PIC/date via lightweight popovers (NOT a Dialog) → Enter. The slim 5-field Dialog ([+ Tugas]) stays for choosing a non-default column; "Detail lengkap…" escalates to the existing full open_task_dialog.
  • Triage (1 click/field, inline, self-emptying): untriaged tasks auto-surface (!assigned_to → Belum Ditugaskan; overdue/no-deadline → Terlambat). Each row renders ONLY the empty chips. Click 👤 → dropdown of board.team → patch_task('assigned_to'); ⚑ → 4-item menu → patch_task('priority'); 📅 → native date → patch_task('deadline'). On success the row re-derives; when it matches no action bucket it animates out (work-down queue).
  • Batch triage: Belum Ditugaskan header [☐ pilih semua] → bulk bar → [Tugaskan ke ▾] → bulk_assign(project_id, ids, user).
  • Unblock: a Diblokir row shows [Buka blok →] only when the target is in that card's allowed_targets → move_task(id, target).
  • Keyboard: n focus capture, / focus search, Enter submit, Esc closes popover / clears selection.
  • Persistence: per-bucket collapse state in localStorage["vt_pd_collapse_<project_id>"].

6. What changes vs current

  • Removed as tabs: 7-tab bar → 6-item segmented switch; Fokus is the new default. Ringkasan is subsumed by the Fokus feed (richer open-task surface); Gantt moves under "⋯".
  • Kept (unchanged code): Papan (kanban), Kalender, Sprint, Milestone, Tim — their render_* fns render verbatim behind the segment switch.
  • Added: Fokus feed, sticky quick-capture, KEPADA load lens, batch-assign bulk bar.
  • Changed behavior: page opens to the attention feed, not the board (deliberate, aligned with leader-oversight thesis). First-run hint via vt_render_empty_state explains the new default.

7. Implementation

Single page file task/page/vt_project_detail/vt_project_detail.js (vanilla JS + jQuery + frappe.ui, no build) + styles appended to public/css/vt_home.css (reuse --vh-* / .vh-chip-* / .vb-prio-* tokens). The existing load_page already does Promise.all(project_detail, project_board) in on_page_show — the feed slots into render_page. Pure helpers (keep handlers logic-free, functions < 40 lines, named constants for bucket keys/order/labels):

  1. flatten_board(board) → all cards.
  2. bucketize(cards, today) → ordered buckets via server overdue + kanban_status + assigned_to + risk_flag + deadline.
  3. render_feed() → collapsible sections; each row a template rendering only-missing chips.
  4. inline chip handlers → patch_task then re-run bucketize + re-render in place (mirror existing reload_board).
  5. assignee_load(cards, team) → KEPADA counts.
  6. quick-capture → one input + 3 popovers + create_task.
  7. bulk bar → selection state + bulk_assign.
  8. Papan/Kalender/Sprint/Milestone/Tim reuse existing render_* behind the segment switch (LAZY_TABS map already exists).

8. Tests

  • task/api/test_*: _map_task_row returns overdue correct across site-tz boundary (use getdate(today()) both sides, build dates via add_days); deadline today → overdue=False, yesterday → True, none → False.
  • test_board_mutations.py: bulk_assign — happy path sets assignee on all ids; rejects a task_id outside the project (_assert_project_access); rejects a non-team assignee; honors rate-limit; permission denied for non-member caller.
  • test_vt_project_detail.py: page still loads; segment list contains Fokus default; existing assertions preserved.
  • Bucketing/derivation are pure JS helpers — assert the contract via the backend overdue field; client helpers covered by the page smoke test + manual verification.

9. Risks & honest caps

  • 200-task board cap (DETAIL_TASK_LIMIT): an overdue/unassigned task beyond the cap is missed (same as today's Ringkasan). Server-side pre-bucketing is the documented later add if a project exceeds 200 open tasks.
  • Date correctness: Terlambat/Due-today MUST use the new server overdue:bool; browser-local Date would mis-bucket near midnight. The backend change is non-optional.
  • bulk_reassign authz: never call portal_projects.bulk_reassign directly from the board — ship the guarded bulk_assign wrapper first.
  • patch_task rate-limit (30/min): one-row-at-a-time bulk triage could brush it; the batch path mitigates; surface a 429 toast.
  • risk_flag is async (stamped by risk_evaluator) — Berisiko is not a live recompute; a freshly-late task still shows under Terlambat.
  • Single-bucket placement: a blocked+overdue task counts once (Diblokir); per-row chips partially compensate. Accepted to keep counts non-overlapping.
  • Board demotion: behavior change for kanban-spatial users; mitigated by one-click Papan + first-run hint.
  • Restart: new bulk_assign whitelist + overdue field need a backend restart (gunicorn imports modules once at worker start).
