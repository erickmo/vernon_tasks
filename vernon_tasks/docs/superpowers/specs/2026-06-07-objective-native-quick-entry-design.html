Objective form — Frappe native quick entry

Date: 2026-06-07 Task size: M (doctype config + 1 page JS + 1 API trim + tests; ~5 files) App ownership: 100% vernon_tasks — no Frappe core files touched Base branch: master Builds on: docs/superpowers/specs/2026-06-04-brand-detail-okr-design.html

1. Goal

On vt-brand-detail, replace the custom frappe.ui.Dialog used to create/edit an Objective with Frappe native flows:

  • Create (+ Objective) → native quick-entry dialog (frappe.ui.form.make_quick_entry).
  • Edit (pencil on a card) → native full Objective form (frappe.set_route("Form", "Objective", id)).

Key Result create/edit keeps its existing inline dialog — out of scope.

This is the Vernon Frappe low-code principle applied: prefer native config/UI over custom code. The Objective controller already owns every domain rule (period auto-fill, owner-enabled check, PDCA state machine, progress rollup), so the native frappe.client.save path validates identically to the old whitelisted mutation — the custom dialog + its mutation endpoints become redundant.

2. Why native is safe (verified)

Frappe 15.107.1. Read of frappe/public/js/frappe/form/quick_entry.js confirms:

  • is_quick_entry() returns false unless meta.quick_entry == 1 → without the flag, make_quick_entry falls back to the full form. We therefore set quick_entry: 1 on the doctype (chosen over force=true so the native desk + New gains quick entry too — the native-config way).
  • Surfaced fields = (df.reqd || df.allow_in_quick_entry) && !df.read_only && !df.is_virtual. The meta flag is allow_in_quick_entry (not in_quick_entry).
  • set_defaults() seeds dialog inputs from the passed doc; update_doc() reads back via dialog.get_values(true) which includes read-only fields → a prefilled + locked brand persists to insert.
  • When an after_insert callback is supplied, the dialog hides and does not redirect to the new form → we stay on the brand page.
  • Objective is not submittable → no Submit step.

3.2 Page JS — task/page/vt_brand_detail/vt_brand_detail.js

// Create — native quick entry, brand-scoped
function objective_create(page, brand_id) {
    frappe.ui.form.make_quick_entry(
        OBJECTIVE_DOCTYPE,                       // "Objective"
        () => load_page(page, brand_id),         // after_insert: refresh, stay on page
        (dialog) => {                            // init_callback: lock brand to this page
            dialog.set_value("brand", brand_id);
            dialog.set_df_property("brand", "read_only", 1);
        },
        { brand: brand_id, objective_owner: frappe.session.user },  // prefill
    );
}

  • Edit wiring (.vt-obj-edit click): frappe.set_route("Form", OBJECTIVE_DOCTYPE, o.id) instead of objective_dialog(... o.id).

4. Edge cases & risks

CaseHandling
PermissionNative frappe.client.save enforces create perm (same as old has_permission check). Button still gated on can_create_objective.
Brand tamperingField prefilled + locked read-only in dialog; value still flows to insert. (A determined user could POST another brand, but that was equally true of any create — the controller validates brand existence.)
Owner requiredPrefilled to session user; user may change. Controller rejects disabled owners.
Period formatHint shown via field description; controller throws on invalid grammar.
"Edit Full Form" linkNative quick entry adds this escape link — acceptable (consistent native UX).
Migrationbench migrate applies the doctype flag inside the container.
