VT Settings Hub + Project Detail + Quick-Create Project

Date: 2026-05-31
Task size: L (2 new desk Pages + new API module + vt-projects update + fixtures + tests; >10 files)
App ownership: 100% vernon_tasks — no Frappe core files touched
Scope: (A) add a "Buat Proyek" quick-create dialog to vt-projects; (B) new vt-project detail page; (C) new vt-settings hub page that edits navbar2 + login branding + scoring with a friendly UX. All pages reuse the vt-home .vh-* style.

1. Goal

Give VT users a desk-native, vt-home-styled flow to: create a project without leaving the projects list, open a project's detail dashboard, and let VT Managers edit VT Settings (navbar2 menu, login branding, scoring) from a friendly page instead of the raw /app form. No Frappe core files touched; native Frappe infrastructure only.

4. API Contracts

dashboard.project_detail(project_id)

Read-only. require_login(); user must lead or be a member of the project (or admin) — else frappe.PermissionError. Returns:

{
  "header": {"id","title","status","pdca_phase","percent_done","risk",
             "leader","start_date","end_date","sprint": {...}|null},
  "open_tasks": [{"id","title","kanban_status","priority","deadline","risk_flag"}],
  "team_members": [{"user","role"}],
  "milestones": [{"title","due_date","status"}],
  "blockers": int
}

settings.get_settings() / save_settings(payload)

Both guard: caller must have role VT Manager (else frappe.PermissionError).

  • get_settings() → {navbar_items:[{label,route,icon,enabled,idx}], branding:{login_headline,login_subtext}, scoring:{weight_multiplier,early_bonus_rate,late_penalty_rate,revision_deduct_rate,default_daily_target_hours}}
  • save_settings(navbar_items, branding, scoring) → loads single VT Settings doc, sets branding + scoring fields, replaces navbar_items child rows (order = list index), .save(). Returns {ok: true}.

Quick-create uses native frappe.db.insert from JS — no custom endpoint.

5. Page Layouts (.vh-* reuse)

vt-project

Hero header card: title, status chip, risk chip, progress bar (vh-bar), leader + dates + pdca_phase meta. Then sections: Task Terbuka (vh-item list), Tim (member list), Milestone (list with due/status). Buttons: Refresh, Edit (native form).

vt-settings

3 vh-section: Navbar (row per item: label/route/icon inputs, enabled toggle, ↑↓ reorder, delete; "+ Tambah" adds row), Branding Login (login_headline, login_subtext), Scoring (5 number fields). Single "Simpan" button → save_settings.

6. Tests

  • Each page: exists / page_name / role gating (pattern of test_vt_projects.py).
  • project_detail: happy-path returns sections; bad/forbidden id raises.
  • settings: get returns shape; save persists; non-manager raises PermissionError.

7. Out of Scope

  • No change to DEFAULT_NAVBAR or vt_navbar.js rendering.
  • No new project fields; quick-create uses existing fields only.
  • No edits to native VT Settings form layout (the doctype form still works as-is).
